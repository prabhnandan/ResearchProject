Specify paths for files:

-f [path of .gxl input]
-o [path of .gxl output]


To create a clustering only, name the clustering algorithm:
-clusterer [name]
e.g. -f "in.gxl" -o "out.gxl" -clusterer dcp
This will output a schedule on unlimited processors.


To create a schedule on limited processors, specify the processor count and the scheduler:

-p [number of processors]

for a list-scheduler, include a priority and placement-scheme:
-priority [name]
-placement [name]
-no-insertion	#disables task insertion which is used by default

for a cluster-scheduler, include a clusterer, cluster-merger, and ordering:
-clusterer [name]
-merger [name]
-ordering [name]

e.g.
-f "in.gxl" -o "out.gxl" -p 5 -priority blevel -placement latest-children-est -no-insertion
-f "in.gxl" -o "out.gxl" -p 10 -clusterer dcp -merger list -ordering est


Here are the names for algorithm parts.

Task priority:
blevel - decreasing bottom level
dls - Dynamic Level Scheduling
etf - Earliest Time First
mcp - Modified Critical Path
cp-tlevel - Critical-Path-Node-Directed
cp-critical-parent - Decisive Path

Placement:
norm - normal way of minimising task's own EST
latest-children-est - minimises latest of ESTs of all children
weighted-children-est - pminimises weighted ESTs of all children
critical-child-est - minimises sum of ESTs of the task and its critical child 
cpop - Critical Path On a Processor

clusterer:
dcp - Dynamic Critical Path
dsc - Dominant Sequence Clustering
lc - Linear Clustering

cluster-merger:
glb - Guided-Load-Balancing
list - merge by list-scheduling

ordering:
blevel - decreasing bottom level
est - increasing EST

Use -help for more details.
